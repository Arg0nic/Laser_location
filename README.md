# Симулятор рабочей дальности лазерного дальномера

Этот проект реализует программную часть моделирования максимальной рабочей дальности лазерного дальномера БПЛА в турбулентной приземной атмосфере.

Приложение представляет собой CLI-инструмент на Python. Оно читает JSON-конфигурацию, строит сетку расстояний, применяет двухэтапный критерий, сохраняет табличные результаты и формирует графики.

## Обзор модели

Симулятор реализует именно тот двухэтапный алгоритм, который задан в постановке задачи.

### Этап 1: геометрическая фильтрация

Основные константы и соотношения:

- `d_target = 0.5 m`
- `R_t = d_target / 2 = 0.25 m`
- `theta_0 = 0.1 mrad = 1e-4 rad`
- `k = floor(L / 200)`
- `theta(L) = theta_0 * 3^k`
- базовая модель пятна: `d(L) = theta(L) * L`
- `R_s(L) = d(L) / 2`

Правило принятия решения:

- если `d(L) > 0.5 m`, расстояние сразу отбрасывается
- для отброшенных расстояний `p(L) = 0`
- метод Монте-Карло запускается только если `d(L) <= 0.5 m`

### Этап 2: моделирование Монте-Карло

Для каждого расстояния `L`, прошедшего геометрическую фильтрацию, программа выполняет `N` испытаний Монте-Карло.

Каждое испытание включает:

1. Блуждание луча
   - генерируются `X, Y ~ Normal(0, sigma_w(L)^2)`
   - вычисляется смещение `r = sqrt(X^2 + Y^2)`
   - в проекте используется линейная модель блуждания:
     `sigma_w(L) = sigma_w_value + sigma_w_slope * L`

2. Доля энергии на цели
   - лазерное пятно и цель рассматриваются как круги
   - площадь перекрытия устойчиво вычисляется для случаев:
     - отсутствия пересечения
     - полного вложения
     - частичного пересечения
     - совпадения центров
   - `eta = S_intersection / (pi * R_s^2)`
   - для вырожденного случая нулевого радиуса пятна в коде есть отдельная обработка

3. Энергетический порог
   - если `eta < eta_min`, испытание сразу считается неуспешным

4. Шумовая амплитуда и EWMA-фильтрация
   - `A_ideal = A0 * eta`
   - для каждого импульса:
     - `A_t = A_ideal + b + epsilon_t`
     - `epsilon_t ~ Normal(0, sigma_A^2)`
   - EWMA:
     - `Ahat_1 = A_1`
     - `Ahat_t = alpha * A_t + (1 - alpha) * Ahat_(t-1)`
   - если итоговая отфильтрованная амплитуда удовлетворяет условию `Ahat_M >= T`, испытание считается успешным

После всех испытаний:

- `p(L) = successes / N`

Итоговая рабочая дальность:

- `L_work = max { L : p(L) >= p_required }`

## Требования

- Python 3
- `numpy`
- `matplotlib`

При необходимости установите зависимости:

```bash
pip install numpy matplotlib
```

## Как запустить

Запуск с конфигурацией по умолчанию:

```bash
python main.py
```

Запуск с явным указанием конфигурационного файла:

```bash
python main.py --config configs/default_config.json
```

Пример с несколькими переопределениями параметров через CLI:

```bash
python main.py --config configs/default_config.json --L-max 1000 --N 4000 --sigma-w-value 0.03 --sigma-w-slope 0.0002
```

Запуск без построения графиков:

```bash
python main.py --no-plots
```

## Параметры конфигурации

Файл JSON по умолчанию содержит основные параметры:

- `L_min`, `L_max`, `dL`: сетка расстояний
- `N`: число испытаний Монте-Карло на каждом допустимом расстоянии
- `M`: число импульсов внутри одного испытания Монте-Карло
- `theta_0`: базовая расходимость в радианах
- `d_target`: диаметр цели в метрах
- `eta_min`: минимальная доля энергии на цели перед переходом к сигнальной части
- `A0`: коэффициент масштабирования идеальной амплитуды
- `b`: постоянное смещение сигнала
- `sigma_A`: среднеквадратичное отклонение шумовой добавки к амплитуде
- `T`: финальный порог детектирования после EWMA
- `alpha`: коэффициент EWMA
- `sigma_w_value`: свободный член линейной модели `sigma_w`
- `sigma_w_slope`: коэффициент наклона в линейной модели `sigma_w`
- `p_required`: минимально допустимая вероятность успешной работы
- `random_seed`: фиксированный seed для воспроизводимости

## Какие файлы создаются

Каждый запуск создает новую директорию с отметкой времени внутри `outputs/` и сохраняет:

- `results.csv`: таблицу результатов по всем расстояниям
- `used_config.json`: точную конфигурацию, использованную в запуске
- `spot_diameter.png`: график `d(L)` от `L`
- `success_probability.png`: график `p(L)` от `L`

CSV-файл включает:

- расстояние
- `theta(L)`
- `sigma_w(L)`
- диаметр и радиус пятна
- флаг прохождения первого этапа
- вероятность успеха `p(L)`

## Краткая сводка в консоли

После каждого запуска CLI выводит краткую информацию:

- основные значения параметров
- максимальную дальность, прошедшую геометрический этап
- рабочую дальность, удовлетворяющую условию `p(L) >= p_required`
- путь к выходной директории и созданным файлам

## Воспроизводимость

По умолчанию симулятор использует фиксированный seed генератора случайных чисел, поэтому повторные запуски с одной и той же конфигурацией дают одинаковые результаты Монте-Карло. При изменении seed или любого параметра модели результат изменится.
